## 摘要
音语言模型可通过提示进行语码转换语音处理，但其解码过程并未针对语码转换优化，常在语言边界处出现错误。我们提出一种实用的、带有可验证奖励的强化学习方法，利用组相对策略优化，结合错误率奖励和惩罚错误书写系统的文本忠实度奖励，以及双阶段草案-精修流程，实现音语言模型对语码转换语音识别的数据高效适配。以Qwen2-Audio为可复现测试平台，在10个语言对上仅使用文本到语音合成的语码转换语音进行训练，我们证明使用10%数据的RLVR能达到在全量数据上训练的LoRA监督微调的性能，且在类型学上差异较大的语言对上提升最大。错误率奖励消除了翻译错误，而文本忠实度奖励在不降低性能的情况下单独减少了书写系统混杂。这些提升在零样本条件下迁移到了人工录制的语码转换语料库。

## Abstract
Audio-language models can be prompted for code-switched speech, but their decoding is not optimized for code-switching and often fails at language boundaries. We propose a practical reinforcement learning with verifiable rewards recipe for data-efficient adaptation of audio-language models to code-switched ASR using group relative policy optimization, combining an error rate reward with a script fidelity reward that penalizes wrong writing systems and a two-pass draft-and-refinement procedure. Using Qwen2-Audio as a reproducible testbed across 10 language pairs, training on only TTS code-switched speech, we show that RLVR with 10% of the data matches LoRA supervised fine-tuning trained on the full dataset, with the largest gains on typologically distant pairs. The error rate reward eliminates translation errors while the script fidelity reward separately reduces script contamination without degradation. These gains transfer zero-shot to a human-recorded code-switching corpus.

*   **整体含义**：本研究将语码转换语音识别（CS-ASR）构建为一个“可验证奖励”问题，旨在探索如何利用强化学习（RL），通过精心设计的奖励函数，以极少的训练数据和计算资源，让模型直接学会优化序列级的转录质量，从而解决上述边界失效问题。

### 2. 论文提出的方法论

*   **核心思想**：使用强化学习与可验证奖励（RLVR），直接优化与转录质量和语码转换忠实度相关的序列级指标，而不是传统的词元级交叉熵损失。
*   **关键技术与算法流程**：
    *   **基础算法：组相对策略优化（GRPO）**：
        *   训练时，对每个音频输入采样**G**个候选转录。
        *   根据可验证奖励函数**R**为每个候选转录打分。
        *   计算组内标准化奖励（Z-score）作为优势函数**A<sub>i</sub>**，鼓励模型增加得分高于组平均的候选转录的概率，反之亦然。
        *   策略通过带有裁剪和KL正则化的替代目标进行更新，无需学习独立的价值函数。
    *   **奖励函数设计**：
        *   **错误率奖励**：基于编辑距离的**字符错误率（CER）** 的负值（-CER），直接衡量转录准确性，能有效消除翻译错误。
        *   **脚本保真度奖励**：一个二元奖励，如果模型生成的所有字符都属于该语言对允许的文字系统（如拉丁字母和中文字符集的并集），则给予`βsf`的正向奖励。此举旨在独立惩罚并减少`脚本污染`。
    *   **双阶段草案-精修流程（Two-Pass Draft-and-Refinement）**：
        *   **第一遍（草案）**：根据原始音频生成**G**个候选转录，用GRPO进行一次更新，并选择奖励最高的草案`ŷ†`。
        *   **第二遍（精修）**：将原始音频、最佳草案`ŷ†`和语言上下文组合成新的提示，再次生成**G**个精修转录，并进行第二次GRPO更新。此过程训练模型学会“再听一遍并修正”，是一种训练时的自修正策略，测试时仍用单遍解码。

### 3. 实验设计

*   **数据集/场景**：
    *   **训练数据**：**CS-FLEURS**的`XTTS-TRAIN`子集（128小时，16个X-英语对的TTS合成语音）。实验从该子集中抽取10%和20%的数据进行数据高效性测试。
    *   **评测数据1**：**CS-FLEURS**的`READ-TEST`子集（3.3小时，14个X-英语对的人工录制的双语语音）。
    *   **评测数据2（零样本迁移）**：**SwitchLingua**，一个超过80小时的人工录制语码转换数据集，包含单轮和多轮对话，用于测试跨域泛化能力。
*   **基准方法**：
    *   **仅格式提示**：仅用格式奖励（要求输出在特定标签内）训练极少数迭代的基础模型，无转录质量信号。
    *   **LoRA监督微调（SFT）**：使用**全量** `XTTS-TRAIN` 数据进行低秩适配（LoRA）监督微调，作为监督学习上界。
*   **主要指标**：
    *   **字符错误率（CER）**：经过小写化、去标点和去空格处理后的编辑距离。
    *   **脚本幻觉率（SHR）**：包含不允许的文字系统字符的样本比例。

### 4. 资源与算力

*   **硬件配置**：所有实验均在 **8块NVIDIA A100 GPU** 上进行，启用bfloat16精度。
*   **训练详情**：

### 6. 论文的主要结论与发现

*   **数据高效性**：**仅用10%的训练数据**，RLVR就能在测试集上取得与**全量数据LoRA SFT**相匹配的性能；使用20%的数据时，性能**超越**了LoRA SFT，且所需的梯度更新步数约为后者的1/40。

    *   **CER奖励**几乎完全消除了翻译错误（将非英语翻译为英语）。
    *   **脚本保真度奖励**是独立且必要的，它在不损害CER的前提下，显著降低了`脚本污染`问题。
*   **泛化能力**：在合成数据（CS-FLEURS）上训练的RLVR模型，其性能优势可**零样本迁移**到人工录制的真实语码转换数据集（SwitchLingua）。
*   **性能分布**：RLVR的最大提升出现在**类型学上差异较大的语言对**（如阿拉伯语-英语、日语-英语），即语码转换难度最大的场景。LoRA在欧洲语言对和高资源语言上仍有微弱优势。

### 7. 优点

*   **创新的方法组合**：首次将GRPO与专门为CS-ASR设计的可验证奖励（CER+脚本保真度）及双阶段精修训练过程相结合，极具针对性。
*   **解决特定问题的精巧奖励设计**：脚本保真度奖励巧妙地利用了可验证的Unicode文字系统信息，直接且有效地打击了`脚本污染`这一具体失败模式。
*   **出色的数据与计算效率**：极少的训练数据和梯度步数即可匹配或超越全量数据微调，为低资源场景下的模型适应提供了极佳的方案。
*   **鲁棒的零样本泛化能力**：从TTS合成数据到真实人类语音的零样本迁移成功，证明了该方法学到的是一些本质的、鲁棒的规律，而非仅仅拟合合成数据分布。

### 8. 不足与局限

*   **模型泛化性未经验证**：所有实验均基于`Qwen2-Audio`一个模型，所提出的方法是否对其他架构的speech-LLMs普遍有效，尚待验证。
*   **评估指标局限性**：CER和SHR可能无法完全捕捉语码转换质量的所有方面。文中也提到SHR奖励在拉丁字母对中会间接提升CER，这暗示了需要更全面的评估指标。
*   **训练效率问题**：尽管数据高效，但RLVR的单步计算成本（需采样G=8个完整序列并计算奖励）远高于SFT，导致在相同数据量下总训练时间更长。
*   **双阶段优化的负面效应**：在阿拉伯语-英语对上，双阶段精修训练反而导致性能下降，表明这种方法可能并非对所有语言对普遍有益，其“过修正”风险需要关注。
*   **LoRA的残留优势**：LoRA在产生“近乎完美”的转录（CER < 0.05）方面仍优于RLVR，说明RLVR可能在精确复制特定技术词汇或高精度匹配上存在短板。
