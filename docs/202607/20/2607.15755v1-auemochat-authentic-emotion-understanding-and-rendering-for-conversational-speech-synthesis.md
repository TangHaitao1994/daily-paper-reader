## 摘要
对话语音合成（CSS）旨在在用户与智能体的交互中合成具有类人情感表达和上下文一致性的语音。现有CSS方法由于受限的预定义情感标签空间（例如七种情绪类别）而难以渲染真实的人类情感，同时多轮对话历史中的冗余多模态令牌会干扰上下文理解。为解决这些问题，我们提出AuEmoChat，一个用于真实情绪理解与渲染的CSS框架。首先，我们开发了AuEmoCodec，它通过有限标量量化从大规模情感语音中学习离散的真实情绪令牌空间，从而比有限的基本情绪类别实现更真实的情绪表示。我们进一步提出AuEmoToMe，一种由真实情绪引导的令牌合并算法，在保留情绪相关上下文的同时合并多模态对话历史中的冗余令牌。我们将其集成到自回归文本-语音模型中，以预测目标真实情绪令牌和语音令牌。最后，我们提出真实情绪流匹配，通过联合条件作用于合并的对话上下文、目标真实情绪和声学先验来渲染语音。在NCSSD-EmCap数据集上的大量实验表明，AuEmoChat优于最先进的CSS基线，并生成更具表现力和真实感的情绪语音。

## Abstract
Conversational Speech Synthesis (CSS) aims to synthesize speech with human-like emotional expression and contextual consistency in user-agent interactions. Existing CSS methods struggle to render authentic human emotions due to limited predefined emotion label spaces (e.g., seven emotion categories), while redundant multimodal tokens in multi-turn dialogue history interfere with context understanding. To address these issues, we propose AuEmoChat, a CSS framework for authentic emotion understanding and rendering. First, we develop AuEmoCodec, which learns a discrete authentic emotion token space from large-scale emotional speech via finite scalar quantization, enabling a more authentic emotion representation than limited basic emotion categories. We further propose AuEmoToMe, an authentic-emotion-guided token merging algorithm that merges redundant tokens in multimodal dialogue history while preserving emotion-relevant context. We integrate it into an autoregressive text-speech model to predict the target authentic emotion token and speech tokens. Finally, we propose Authentic Emotion Flow Matching, which renders speech by jointly conditioning on merged dialogue context, target authentic emotion, and acoustic priors. Extensive experiments on the NCSSD-EmCap dataset demonstrate that AuEmoChat outperforms state-of-the-art CSS baselines and generates more expressive and authentic emotional speech.

基于此，论文提出**AuEmoChat框架**，核心目标是实现对对话中“真实情绪”（Authentic Emotion）的深层理解与高保真渲染，使智能体能够合成更具表现力和沉浸感的语音回应。

### **2. 论文提出的方法论**

AuEmoChat 的整体架构包含三大核心模块（图2左侧）：多模态对话令牌化、基于 AuEmoToMe 的真实情绪理解、以及基于合并上下文的真实情绪渲染。

-   **AuEmoCodec：学习真实情绪令牌空间**
    -   **核心思想**：摒弃传统离散标签，直接从大规模情感语音数据中学习一个能表达细腻情绪的离散特殊令牌——**AuEmo Token**。
    -   **技术细节**：
        1.  使用语音情绪编码器和**有限标量量化（FSQ）** 技术，将情绪语音压缩为一个离散码本中的索引（AuEmo Token，码本大小1000）。
        2.  训练时，不重构原始波形，而是利用 Gemini-2.5-Flash 模型为语音在七个基本情绪轴上标注的“感知分”作为监督信号，让解码器从 AuEmo Token 重构这些分数。
    -   **功能**：训练完成后，AuEmo Token 能够表征远超七种基本情绪的、细腻连续的“真实情绪状态”。

-   **AuEmoToMe：真实情绪引导的令牌合并算法**
    -   **核心思想**：在自回归（AR）建模前，智能合并对话历史中的冗余文本和语音令牌，以减少干扰、保留关键信息。
    -   **技术细节（算法1）**：
        1.  **分割与匹配**：对每轮对话的文本/语音令牌序列（源集和目标集），按给定比例（如30%）计算余弦相似度，找到最相似令牌对。
        2.  **情绪引导加权融合**：在合并一对令牌时，计算它们与同一话语中 AuEmo Token 的相似度，并将其作为权重进行加权聚合。这确保与情绪相关的关键令牌信息在合并过程中被优先保留。
        3.  **输出**：生成一个压缩且富含情绪信息的“合并后对话历史表示”。

-   **真实情绪理解与推理**
    -   将合并后的对话历史令牌序列与目标话语的说话人、文本令牌拼接，输入一个自回归模型。
    -   模型首先推理出当前话语应具备的**真实情绪令牌**，再以此和上下文为条件，自回归地生成目标语音的**声学令牌序列**。

-   **真实情绪流匹配渲染**
    -   **核心思想**：在生成最终的语音频谱时，联合约束声学路径、合并的对话上下文和预测的目标真实情绪。
    -   **技术细节**：
        1.  **条件编码**：通过 GRU 编码器处理合并的上下文令牌，通过情绪编码器处理预测的 AuEmo Token，获取上下文条件`F_ctx`和情绪条件`F_e`。
        2.  **流匹配**：以预测的声学令牌为先验`μ`，用`F_ctx`和`F_e`作为条件，引导流匹配网络学习从高斯噪声`x_0`到真实梅尔频谱`x_1`的最优传输路径。
        3.  **分类器引导**：引入一个预训练的 AuEmo 分类器，在流匹配后期（`t > 0.7`）对中间状态提供梯度，引导其朝向更贴合目标情绪的方向演化，最终生成梅尔频谱，并转换为波形。
