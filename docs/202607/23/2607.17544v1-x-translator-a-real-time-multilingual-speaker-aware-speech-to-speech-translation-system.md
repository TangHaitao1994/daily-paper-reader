## 摘要
实时语音到语音翻译（S2ST）系统必须在翻译质量、延迟、语音自然度和说话人一致性之间取得平衡。公开记录的 S2ST 系统已经在直接、多语言、流式和表达性建模方面取得了进展，而专有产品和 API 越来越多地向用户提供实时翻译能力。然而，对于开放和可复现的系统，实际部署仍然具有挑战性，尤其是在长会话和多说话人对话中，部分 ASR 假设不稳定，轮次边界模糊，且目标语音必须使用合适的说话人提示生成。我们提出了 X-Translator，一个低成本的模块化级联 S2ST 系统，通过会话级运行时控制器结合了流式 ASR、机器翻译和提示条件 TTS。该系统使用增量片段提交将不稳定的 ASR 流转换为翻译就绪单元，并使用在线说话人提示管理器将源语音片段绑定到说话人特定的语音提示以进行合成。我们使用 OpenSTBench 评估翻译、语音质量和延迟，与专有语音翻译 API 进行比较作为行为基线，测量长会话语音稳定性，评估多说话人对话中的说话人保留，并评估多语言翻译质量。X-Translator 为理解面向部署的 S2ST 的实际权衡提供了一个开放平台。代码和演示可在 https://github.com/zhaoyx239/X-Translator 获取。

## Abstract
Real-time speech-to-speech translation (S2ST) systems must balance translation quality, latency, speech naturalness, and speaker consistency. Publicly documented S2ST systems have advanced direct, multilingual, streaming, and expressive modeling, while proprietary products and APIs increasingly expose real-time translation capabilities to users. However, practical deployment remains challenging for open and reproducible systems, especially in long-form and multi-speaker conversations where partial ASR hypotheses are unstable, turn boundaries are ambiguous, and target speech must be generated with an appropriate speaker prompt. We present X-Translator, a low-cost modular cascaded S2ST system that combines streaming ASR, machine translation, and prompt-conditioned TTS through a session-level runtime controller. The system uses incremental segment commitment to convert unstable ASR streams into translation-ready units, and an online speaker prompt manager to bind source speech spans to speaker-specific voice prompts for synthesis. We evaluate translation, speech quality, and latency with OpenSTBench, compare against proprietary speech translation APIs as behavioral baselines, measure long-form voice stability, evaluate speaker preservation in multi-speaker conversations, and assess multilingual translation quality. X-Translator provides an open platform for understanding the practical trade-offs of deployment-oriented S2ST. Code and demo are available at https://github.com/zhaoyx239/X-Translator.

    -   **部署难题**：在实际的长时、多说话人对话场景中部署级联系统，存在三大具体难点：

        3.  **长期声纹稳定性**：如何在长时间会话中保持合成目标语音的说话人一致性，并忠实还原源说话人的音色。

### 2. 论文提出的方法论

X-Translator的核心方法论在于其**模块化、可替换的级联架构**和两个关键的**运行时控制机制**。它本身不提出新的ASR、MT或TTS模型，而是专注于如何将它们高效、稳定地组合起来。

-   **核心思想**：采用`流式ASR → 机器翻译 → 提示条件TTS`的级联范式，并在其中间插入一个“会话控制器”层，将翻译任务从连续的音频流中解耦成稳定的、归属明确的文本片段进行批量处理。

-   **关键技术细节**：
    -   **系统架构**：浏览器端通过WebSocket将流式音频和会话控制发送至服务器。服务器端的会话控制器维护ASR假设缓冲区、已提交片段、说话人提示状态和前后端事件队列。
    -   **增量片段提交**：
        -   **核心问题**：区分“展示用”和“翻译用”的ASR结果。网页端可实时显示不稳定部分，但下游翻译只接收稳定的、已提交的前缀片段。
        -   **提交触发机制**：控制器维护一个增长中的源文本缓冲区，并根据以下三种运行时触发器剪切出稳定的前缀：
            1.  **标点触发**：当ASR输出出现强烈的标点边界（如句号）时。
            2.  **静默/无更新触发**：当检测到说话人停顿或ASR流在配置的时间间隔内不再变化时。
            3.  **说话人边界触发**：当检测到说话人切换时，立即清空当前缓冲区，防止一个翻译片段混杂两个说话人的内容。
        -   **数据格式**：每个提交的片段都包含源文本、时间戳、说话人ID等结构化信息，成为下游处理的原子单元。该设计将延迟-质量权衡显式化：激进提交减少延迟但可能失去上下文，保守提交则相反。
    -   **在线说话人提示管理**：
        -   **核心问题**：在多说话人输入中，为每个翻译片段准确分配源说话人，并为其合成目标语音准备高质量音色提示。
        -   **说话人标记与匹配**：使用`CAM++`模型在整个音频时间线上实时提取说话人嵌入并标记活动区间。当一个ASR片段提交时，控制器通过**计算该片段的时间戳与所有说话人活动区间的最大交叠**，来确定片段归属。
        -   **提示桶维护**：系统为每个在线说话人独立维护一个“提示桶”，包含两个6秒的音频视图：
            1.  **固定提示**：由该说话人**前6秒**的有效语音构成，一旦初始化就不变，提供全程稳定的声纹参考。
            2.  **滚动提示**：由该说话人**最近6秒**的有效语音构成，持续更新，以捕捉更即时的发音特征。
    -   **翻译与语音合成**：翻译模块处理提交的稳定文本，合成模块则接收翻译文本和选定的说话人提示音频，生成克隆音色的目标语音。这一流程保证了严格串行，避免了ASR回溯导致的重复播放。

### 8. 不足与局限

-   **级联系统的固有缺陷**：
    -   **误差传播**：ASR的识别错误会直接传递并放大到MT和TTS模块。
    -   **无法联合优化**：各组件独立训练，无法端到端地优化翻译质量、韵律和声纹相似度的复杂权衡，这依赖于人为设定的片段提交策略。
-   **说话人机制的鲁棒性有限**：在线说话人跟踪在重叠语音、快速抢话、噪音环境或声线相似的场景下仍可能出错。一旦前期说话人分配错误，且错误的提示被存入提示桶，将影响后续所有合成片段。

-   **翻译质量与延迟的短板**：在纯粹的性能指标上，X-Translator并未超越顶尖的专有系统。其价值更多体现在提供了可控性与可解释性，而非追求绝对的第一名。此外，多语言评估的COMET分数存在较大标准差，表明其性能在不同语言对上表现并不均匀，这受限于所选用的后端模型能力。
