## 摘要
我们描述了提交至第二届MLC-SLM挑战赛任务1的系统：一个级联的“先说话人分离后识别”系统，结合了DiariZen-Large-s80 (WavLM-Large) 分割、基于CAM++嵌入的两人聚类以及经过LoRA适配的omniASR LLM 7B v2识别器，测试时不使用任何预言分割或说话人标签。在官方开发集（150个对话，21个语言/口音类别）上，该系统获得了29.27%的宏平均tcpMER，而官方基线为79.15%；在评估集上得分为50.23%。我们还分析了两个显著影响tcpMER的工程选择。首先，基于嵌入的说话人聚类优于仅依赖ASR <sc>轮次标记来分配说话人的端到端风格替代方案。其次，重叠感知分割虽然旨在提高说话人分离召回率，但会增加tcpMER，因为重叠语音被转录了两次。

## Abstract
We describe our submission to Task 1 of the 2nd MLCSLM Challenge: a cascaded diarization-then-recognition system that combines DiariZen-Large-s80 (WavLM-Large) segmentation, CAM++ embedding-based two-speaker clustering, and a LoRA-adapted omniASR LLM 7B v2 recognizer, with no oracle segmentation or speaker labels at test time. On the official Development set (150 conversations, 21 language/accent categories) the system attains a macro tcpMER of 29.27%, versus 79.15% for the official baseline; on the Evaluation set it scores 50.23%. We also analyze two engineering choices that substantially affect tcpMER. First, embedding-based speaker clustering outperforms an end-to-end-style alternative that assigns speakers from ASR <sc> turn markers alone. Second, overlap-aware segmentation, although intended to raise diarization recall, increases tcpMER because overlapped speech is transcribed twice.

### 2. 论文提出的方法论
**核心思路**：采用模块化、可拆解的级联管道，将问题分解为前端说话人分割与聚类、后端多语种识别及后处理。

**关键技术细节与流程：**
- **语音活动检测 (VAD)**：使用 FunASR 的 FSMN-VAD 模型，作为所有配置的共同前端。
- **说话人分割**：对比了两种分割器，最终选用基于 WavLM-Large 和 Conformer 端到端分割的 DiariZen-Large-s80 模型。原始片段经过缩短、合并与长片段切分后处理。
- **说话人分配**：
  - 使用预训练的 CAM++ 模型提取 192 维说话人嵌入，在片段子窗上以 0.75 秒步长提取特征。
  - 通过谱聚类将嵌入强制分为 2 个说话人（匹配电话对话场景），并对重叠区域进行后处理以消除时间冲突。
- **语音识别**：
  - 采用开源的 omniASR LLM 7B v2 (基于 wav2vec2 的编码器 + CTC 头) 作为基座模型。
  - 使用 **LoRA** 进行领域适配（秩 r=8, α=16），仅更新低秩适配器，冻结声学编码器，训练目标为 CTC。适配数据来自训练集参考分段（仅训练时可用），并在相邻不同说话人片段间插入 `<sc>` 标记，使模型学会预测话轮边界。
  - 推理时使用语言约束解码，限制输出到指定语言的 token 空间。
- **后处理**：按 `<sc>` 标记切割片段并按字符长度比例分配时间戳；对片段内的近似重复假设进行去重。
- 整体管道从原始音频输入，经 VAD、分割、嵌入聚类、ASR 识别、后处理，输出带有说话人标签和时间点的转录文件。

### 6. 论文的主要结论与发现
- **系统有效性**：提交的级联系统在开发集上达到 29.27% 宏平均 tcpMER，相对官方基线（79.15%）相对降低约 63%，性能提升巨大。
- **说话人分配方法**：基于 CAM++ 嵌入的谱聚类远优于仅依赖 ASR 输出 `<sc>` 标记进行交替分配的方案（Exp1 141.2% vs. Exp3 30.6%），表明可靠的说话人嵌入对分配至关重要。
- **重叠处理策略**：在所采用的 tcpMER 评估指标下，启用重叠感知分割（Exp2）会导致重叠语音被转录两次，增加插入错误，反而恶化整体得分（35.3% vs. Exp3 的 30.6%）。因此，评估综合性的 tcpMER 时，应禁用重叠感知以优化指标。
- **分割器选择**：DiariZen-Large-s80 在关闭重叠后，能提供比 pyannote 分割器更优的段落质量，进一步提升最终性能（29.27%）。
