## 摘要
实时语音到语音翻译（S2ST）系统必须在翻译质量、延迟、语音自然度和说话人一致性之间取得平衡。公开记录的S2ST系统已在直接、多语言、流式和表达性建模方面取得了进展，而专有产品和API越来越多地向用户提供实时翻译能力。然而，对于开放和可复现的系统而言，实际部署仍然具有挑战性，特别是在长对话和多说话人对话中，部分ASR假设不稳定，话轮边界模糊，并且必须使用适当的说话人提示生成目标语音。我们提出X-Translator，一种低成本的模块化级联S2ST系统，通过会话级运行时控制器结合流式ASR、机器翻译和提示条件的TTS。该系统使用增量片段提交将不稳定的ASR流转换为可翻译单元，并使用在线说话人提示管理器将源语音片段绑定到特定说话人的声音提示进行合成。我们使用OpenSTBench评估翻译、语音质量和延迟，与专有语音翻译API进行行为基线比较，测量长对话语音稳定性，评估多说话人对话中的说话人保留，并评估多语言翻译质量。X-Translator为理解面向部署的S2ST的实际权衡提供了一个开放平台。代码和演示可在https://github.com/zhaoyx239/X-Translator获取。

## Abstract
Real-time speech-to-speech translation (S2ST) systems must balance translation quality, latency, speech naturalness, and speaker consistency. Publicly documented S2ST systems have advanced direct, multilingual, streaming, and expressive modeling, while proprietary products and APIs increasingly expose real-time translation capabilities to users. However, practical deployment remains challenging for open and reproducible systems, especially in long-form and multi-speaker conversations where partial ASR hypotheses are unstable, turn boundaries are ambiguous, and target speech must be generated with an appropriate speaker prompt. We present X-Translator, a low-cost modular cascaded S2ST system that combines streaming ASR, machine translation, and prompt-conditioned TTS through a session-level runtime controller. The system uses incremental segment commitment to convert unstable ASR streams into translation-ready units, and an online speaker prompt manager to bind source speech spans to speaker-specific voice prompts for synthesis. We evaluate translation, speech quality, and latency with OpenSTBench, compare against proprietary speech translation APIs as behavioral baselines, measure long-form voice stability, evaluate speaker preservation in multi-speaker conversations, and assess multilingual translation quality. X-Translator provides an open platform for understanding the practical trade-offs of deployment-oriented S2ST. Code and demo are available at https://github.com/zhaoyx239/X-Translator.

### 2. 方法论
系统架构为级联式：**流式 ASR → 机器翻译（MT） → 提示条件文语转换（TTS）**，核心创新在于会话级运行时控制：

- **增量片段提交（Incremental Segment Commitment）**
  - 分离“展示”与“提交”：流式 ASR 的中间结果仅用于前端展示，不直接送入下游；只有稳定片段被“提交”后才会触发翻译和合成。
  - 提交触发策略支持三种规则：① 标点边界 ② 静音或无更新超时 ③ 检测到说话人变更时立即刷新当前缓存。
  - 每个提交片段是结构化事件：源文本、起止时间、说话人 ID 和诊断元数据，保证下游处理单元可控、可追踪。

- **说话人提示管理器（Speaker Prompt Manager）**
  - 使用 CAM++ 说话人嵌入在线标记说话人活动区间，并与 ASR 提交片段的时间跨度进行最大重叠匹配，为每个片段指派说话人。
  - 每名说话人维护一个“提示桶”，包含两个 6 秒提示视图：
    - **固定提示**（fixed prompt）：从该说话人首次可用语音的前 6 秒提取，保持不变，提供稳定的音色参考。
    - **滚动提示**（rolling prompt）：来自该说话人最近 6 秒语音，持续更新以跟踪声学特性的动态变化。
  - 合成时，根据配置选择固定或滚动提示，与翻译文本一起送入 TTS；若提示不足则回退到默认音色。

- **模块化后端**
  - ASR：Qwen3-ASR（1.7B，流式）
  - MT：LMT‑60‑8B（最大 256 tokens 输出）
  - TTS：X‑Voice（0.4B，48kHz 提示条件合成）
  - 所有模块均可替换，且 ASR、MT、TTS 独立通过本地 HTTP 服务调用，系统本身无请求缓存或批处理。

实验分为四大部分，覆盖短句、长对话、多说话人和多语言场景：

- **说话人保留**：X‑Translator 在所有场景（短句、长对话、多说话人）中均取得最高或并列最高的说话人相似度，证明提示条件 TTS 加在线说话人路由能有效保留源说话人特征。
- **语音质量**：在中文→英方向上，目标端词错率（WER）最低（3.04%），合成语音的识别准确性突出。
- **翻译质量与延迟权衡**：翻译质量（COMET）略低于部分商用系统，尤其是英→中方向；但延迟显著偏高（Start 延迟约 2.5–4.1 秒），反映出级联方式在速度和翻译准确度上与端到端/高度优化商业系统的差距。
- **长对话稳定性**：X‑Translator 合成语音的内部一致性（TTLG）接近源语音的自然长间隔一致性，且全局源相似度（GSS）与 Doubao 持平，优于其他基线，表明长期会话中不会出现显著音色漂移。
- **多说话人纯度**：在源纯度和目标纯度上，X‑Translator 与 Doubao 的差距在 0.02–0.07 之间，优于 Qwen 和 GPT，说明其说话人路由能较好地维持多说话人结构，避免明显的说话人崩塌或碎片化。
- **多语言覆盖**：70 个方向的平均 COMET 为 0.69，各组间差异较小，表明框架在多语言场景下具备基础的泛化能力。

### 8. 不足与局限
- **级联错误传播**：ASR 错误不可避免地传导至 MT 和 TTS，且组件间未联合优化，翻译质量和延迟受制于流水线结构。
