## 摘要
强大的语音到文本（S2T）大语言模型已能提供稳健的语音感知与文本推理，但添加语音到语音（S2S）输出仍具挑战：微调骨干网络可能损害原有的S2T性能，而外接下游说话器则重新引入串行的文本到语音瓶颈。我们提出PRIME-Speech，一个冻结骨干的S2S转换框架，仅训练语音生成模块。PRIME-Speech将因果音频后解码器与冻结骨干的中间隐状态同步，使得编解码标记从模型的渐进推理轨迹中生成，而非来自已完成的文本片段。该后解码器使用混合隐状态、文本与音频历史条件，并通过训练时的打包策略配合轮次级音频KV缓存与位置重置，在无需额外多轮S2S训练数据的情况下稳定多轮口语交互。多标记预测进一步降低有效编解码预测率，并在不改变推理路径的前提下改善首音频延迟。在语音翻译、口语问答、语音理解及多轮对话中，PRIME-Speech保持了冻结骨干的S2T行为，同时生成准确、低词错率的口语回应。

## Abstract
Strong speech-to-text (S2T) LLMs already provide robust speech perception and text reasoning, but adding speech-to-speech (S2S) output is challenging: fine-tuning the backbone can degrade the original S2T performance, while attaching a downstream talker reintroduces a serial text-to-speech bottleneck. We present PRIME-Speech, a frozen-backbone S2S conversion framework that trains only speech-generation modules. PRIME-Speech synchronizes a causal audio post-decoder with intermediate hidden states of the frozen backbone, so codec tokens are generated from the model's evolving reasoning trajectory rather than from completed text chunks. The post-decoder uses mixed hidden-state, text, and audio-history conditioning, and a training-time packing strategy with turn-level audio KV-cache and position reset stabilizes multi-turn spoken interaction without additional multi-turn S2S training data. Multi-token prediction further reduces the effective codec prediction rate and improves first-audio latency without modifying the reasoning path. Across speech translation, spoken QA, speech understanding, and multi-turn dialogue, PRIME-Speech preserves the S2T behavior of the frozen backbone while producing accurate, low-WER spoken responses.

### 1. 论文的核心问题与整体含义

- **核心问题**：强大的语音到文本（S2T）大语言模型（LLM）已经具备出色的语音感知与文本推理能力，但将其扩展为语音到语音（S2S）模型时面临两难困境：
  - 若微调整个骨干网络，会破坏原有的 S2T 功能（灾难性遗忘），损害推理与理解能力。
  - 若在骨干后串联一个文本到语音（TTS）模块，则会重新引入串行文本瓶颈与延迟，失去端到端同步生成的优势。
- **整体含义**：论文旨在探索如何在不损害冻结 S2T 骨干能力的前提下，通过附加可训练的语音生成模块，将 S2T LLM 高效地转化为 S2S 模型，从而实现“保留推理、添加语音”的目标。

### 2. 论文提出的方法论

PRIME‑Speech 框架的核心思想是冻结骨干、仅训练语音生成模块，并通过“隐藏状态同步”将语音生成与文本推理实时对齐。

#### 2.1 整体架构与同步机制
- **冻结骨干**：保留原始 S2T 模型（编码器、投影层、Transformer 主干、文本头）的所有参数，不进行任何更新。
- **可训练音频后解码器**：一个因果 Transformer（约 2B 参数），与冻结骨干的中间层隐状态同步。在每一个解码时间步 $s$，骨干输出隐状态 $\mathbf{h}^{\text{mid}}_s$，同时驱动文本头（预测一个文本 token）和音频后解码器（预测一段编码 token 块）。
- **同步方式**：不是等待完整文本响应或固定文本块，而是每步文本与音频分支并行运行，音频分支仅依赖当前隐状态及已提交的历史信息（文本与音频 token），实现时间步级别的实时同步。

#### 2.2 混合条件与因果依赖
音频后解码器的输入是由三个信号加权组合而成（权重均设为 1.0）：
1. **同步骨干状态** $\mathbf{h}^{\text{mid}}_s$：提供语义轨迹。
2. **前一个文本 token 的嵌入** $\mathbf{e}^\tau_{s-1}$：提供词汇锚定。
3. **前一个音频块 token 的平均嵌入** $\mathbf{r}^{a}_{s-1}$：保持局部声学连续性。
训练时采用教师强制，文本历史和音频历史均使用真实参考序列（屏蔽当前步的未来信息），避免对并发文本 token 的依赖。

#### 2.3 多标记预测（MTP）
在音频后解码器之上附加 MTP 头，每个解码步输出 $k$ 个未来编码 token 的概率分布。这使有效编码预测率从 25Hz 降至 25/$k$ Hz，在不改动推理路径的情况下显著降低首音频延迟和实时因子。

#### 2.4 多轮对话的缓存与位置重置
- **训练时**：将不相关的单轮样例打包成伪多轮对话，使音频分支接触长文本上下文和显式轮次边界。
- **缓存策略**：文本 KV 缓存跨轮累积以保持对话语义；音频 KV 缓存每轮重置，避免跨轮声学状态导致的重复或漂移。
- **位置编码**：音频 token 位置按轮次局部重置，文本 token 位置保持全局递增。
- 该策略无需额外多轮 S2S 训练数据，即可利用骨干已有的文本多轮能力。

### 6. 论文的主要结论与发现
1. **能力保留**：PRIME‑Speech 在 S2T 模式下的指标与冻结骨干几乎一致，确认语音生成分支未损害原模型的语音理解与推理。
2. **高质量语音输出**：在所有 S2S 任务上，生成的语音经转录后保持了与文本响应相近的任务正确率，且渲染 WER 低至 3–6%，优于多数对比系统。
3. **隐藏状态同步有效**：无需等待完整文本即可生成准确、流畅的语音，避免文本瓶颈。
4. **MTP 是高效的效率适配器**：将编码预测率降至 6.25Hz 后，首音频延迟和 RTF 大幅降低，任务质量基本持平。
5. **多轮缓存策略关键**：仅需文本缓存累积和音频缓存重置，即可在不使用额外多轮 S2S 数据的情况下实现稳定的多轮对话，避免声学状态污染。

### 7. 优点
- **严格的骨干冻结**：从原理上杜绝灾难性遗忘，提供了一种可扩展的 S2S 转换范式。
- **隐藏状态同步设计**：使语音生成直接基于模型内部推理轨迹，而非成品文本块，兼顾了实时性与语义保真度。
- **简单有效的混合条件**：通过固定权重融合三种信息，无需复杂对齐或额外标注。
- **低成本多轮支持**：仅靠打包策略与缓存重置就激活了骨干的多轮能力，极大降低了多轮 S2S 的数据需求。
- **MTP 渐近升级**：先训练单 token 分支，再引入多 token 预测，使 MTP 成为纯粹的效率组件，不会扰乱已学到的对齐。
- **全面实验对比**：同时报告 S2T 和 S2S 结果，验证了“保留 S2T 能力”与“减小模态差距”这两个核心目标。

### 8. 不足与局限
- **目标语音全部为合成语音**：训练采用 TTS 生成的目标音频，这可能导致模型对真实口语中不流畅性、情感变化、非语言声学特征的适应性不足，在真实人声交互中性能可能下降。
- **任务与语言覆盖有限**：评估集中在英文目标侧的任务，未充分展示对多语种语音输出的支持；多轮对话测试集规模较小（179 轮），缺乏大规模开放域对话验证。
- **计算资源未披露**：未给出训练耗时和硬件需求，难以评估实际训练成本。
- **效率仍非最优**：虽然 MTP 大幅降低了延迟，但与最轻量的专用 talker（如 VocalNet）相比，总体吞吐量和实时因子仍有差距，在极度注重效率的场景下可能受限。
- **未涉及说话人一致性、韵律表现**：论文主要关注任务正确性和渲染一致性，未对说话人身份保持、表现力韵律进行深入评估，而这对真实 S2S 助手应用至关重要。
- **骨干冻结策略的局限**：完全冻结骨干意味着无法利用 S2S 联合训练对语音感知的潜在增强，且中间层状态可能并非最优的语音生成条件。
