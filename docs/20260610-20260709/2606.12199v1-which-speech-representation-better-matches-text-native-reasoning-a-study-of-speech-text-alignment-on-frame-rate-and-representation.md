## 摘要
口语对话模型通常以文本大语言模型为骨干，然而当以语音而非文本为条件时，推理能力往往会下降。我们将这种模态差距部分归因于时间粒度不匹配：在相同语义下，语音token存在时间冗余，且比文本长得多，这稀释了每个token的语义密度，削弱了文本原生的推理动态。我们将语音token设计视为一个表征选择问题，并在固定信息率下，在冻结的LLM骨干上遍历帧率。为了使低帧率可行，我们引入了分解式有限标量量化（FSQ）以及一个轻量级非自回归音频语言模型头，将容量扩展至近300比特/帧，同时不牺牲高效预测。移除瓶颈后，我们遍历了帧率（50→2.08 Hz）和对齐深度，观察到在4.17 Hz帧率与中间层表征对齐下，语音问答一致地达到最佳效果。

## Abstract
Spoken dialogue models typically start from text LLM backbones, yet reasoning often degrades when conditioning on speech instead of text. We attribute part of this modality gap to a temporal-granularity mismatch: speech tokens are temporally redundant and far longer than text under matched semantics, diluting per-token semantic density and weakening text-native reasoning dynamics. We study speech token design as a representation selection problem and sweep frame rates under a frozen LLM backbone with a fixed information rate. To make low frame rates feasible, we introduce factorized FSQ and a lightweight non-autoregressive audio LM head, scaling capacity to nearly 300\,bits/frame without sacrificing efficient prediction. With the bottleneck removed, we sweep frame rates (50$\rightarrow$2.08\,Hz) and alignment depth, and observe a consistent best regime for speech QA at 4.17\,Hz with intermediate-layer representation alignment.

- **研究动机**：在基于文本大语言模型（LLM）构建口语对话系统时，直接以语音 token 替代文本 token 会导致推理性能显著下降，形成模态差距。
- **核心观察**：现有语音 token 化器通常工作在 12.5–50 Hz，而相同语义下文本 token 率平均仅约 3.32 Hz。这种**时间粒度不匹配**使得语音序列过长、单个 token 语义密度被稀释，破坏了 LLM 在预训练中习得的文本原生推理动态。
- **整体思路**：将语音 token 设计视为一个**表征选择问题**，并通过冻结文本 LLM 来隔离变量，系统研究**帧率**和**表征对齐深度**如何影响跨模态推理迁移，旨在找到最接近文本 token 行为、最能继承 LLM 推理能力的语音表示。

### 6. 论文的主要结论与发现
- **帧率最优区间**：在固定信息率下，语音 QA 的最优帧率出现在 **4.17 Hz 和 6.25 Hz**，而非最高或最低帧率。过高导致序列冗余，稀释注意力；过低则因每 token 信息密度过高而损害生成质量，形成非对称 U 型曲线。
- **中间层对齐最有效**：在 LLM 的**中间层**（\(L/2\)）进行表征对齐，比无对齐提升 7.4 个 QA 点；嵌入层对齐则略有下降，深层对齐次优。中间层携带最适于跨模态迁移的语义表示。
- **长度与表征的协同效应**：4.17 Hz 仅做长度对齐得 23.3 分，叠加中间层对齐后达 30.7 分（相对提升 32%），二者互补。
- **与文本 token 率的微幅偏移需求**：最优帧率略高于平均文本 token 率（3.32 Hz），原因是帧率需为整个语料库的语速波动提供缓冲，保持在 LLM 长度容忍窗口（0.8×–2.2× 文本长度）以内。
- **Factorized FSQ 解决了低帧率瓶颈**：在 2.08 Hz 下仍能维持可接受的 ASR 性能，远优于固定小码本基线。
- **泛化性**：框架适用于多种语音编码器（Whisper、HuBERT、WavLM），且提出的语音 token 在重建质量上优于 CosyVoice2。

### 7. 优点
