## 摘要
虽然大型多模态模型在理解方面表现出色，但高通量推理引擎缺乏对多模态生成的原生支持。这在语音语言模型中尤为严重，因为通过解耦的自回归/非自回归或带有延迟模式交织的同步多令牌预测来生成多层音频令牌，与标准的单流循环相冲突。我们提出了一种基于vLLM的推理管线，用于统一的语音理解与生成。我们扩展了自回归解码，以原生执行延迟模式解交织和协调的多流采样，并集成了片上GPU声学解码器，实现端到端波形合成。关键的是，我们克服了无分类器引导会将吞吐量减半的普遍认知。通过在一个连续批次中协同调度配对的条件和无条件请求，我们的无分类器引导实现维持了无引导时80%的吞吐量，吸收了双重请求和logit合并的开销。我们将框架开源。

## Abstract
While Large Multimodal Models excel in comprehension, high-throughput inference engines lack native support for multimodal generation. This is severe in Speech Language Models, where generating multi-layered audio tokens via decoupled AR+NAR or synchronous Multi-Token Prediction (MTP) with delay-pattern interleaving conflicts with standard single-stream loops. We present a vLLM-based inference pipeline for unified speech understanding and generation. We extend autoregressive decoding to natively execute delay-pattern de-interleaving and coordinated multi-stream sampling, integrating an on-GPU acoustic decoder for end-to-end waveform synthesis. Crucially, we overcome the shared intuition that Classifier-Free Guidance (CFG) halves throughput. By co-scheduling paired conditional and unconditional requests within a continuous batch, our CFG implementation sustains 80% of non-CFG throughput, absorbing dual-request and logit merging overheads. We open-source our framework.

- **整体含义**：该工作旨在填补文本中心推理引擎处理多模态自回归生成的能力缺口，通过扩展 vLLM 原生支持语音理解与生成，并设计高效的 CFG 机制，为实时多模态语音应用提供高通量、低开销的部署基础。

### 2. 论文提出的方法论：核心思想与关键技术细节

- **整体架构**：基于 vLLM 连续批处理引擎，扩展自回归解码以原生支持同步多流采样与延迟模式解交织，并在输出管线中融合片上声学解码器，实现端到端波形合成。

- **多令牌预测与主辅流分解（Primary-Auxiliary Decomposition）**
  - 将多个并行码本流中的一个指定为“主流”（primary），由引擎标准管线管理；其余辅助流在模型内部采样并缓冲。
  - 外观上模型仍表现为每步生成单个令牌，实际透明执行多码本生成，避免对引擎调度器与 PagedAttention 的侵入性修改。
  - 请求完成后，缓冲的多流令牌矩阵经延迟解交织，传入共置的片上声学解码器合成波形。

- **混合模态相位管理**
  - 利用动态词汇掩码实现每个请求的相位状态机，依次切换：文本相（仅文本令牌）、过渡相（注入确定性桥接令牌）、音频相（激活所有码本流）。
  - 音频相结束时进入“排空子相”，补发填充令牌以冲洗延迟缓冲区，保证解交织所需完整码本矩阵。
  - 相位判断直接利用模型未掩码的输出分布（对大词汇分区的 argmax），无需依赖输入位置，适配连续批处理中的令牌交织。

- **无分类器引导（CFG）的配对请求共调度（Paired Request Co-Scheduling）**
  - 当请求 CFG 尺度 w>1 时，自动创建“伴随请求”（条件为无文本提示），与“主请求”原子配对。
  - 两者在同一次连续批次中共调度，共享Transformer主干的一次前向传播。
  - 每步 token 预算严格对半分配；仅在音频相合并 logit：\(\tilde{\ell}^{(i)} = w \cdot \ell^{(i)}_{\text{cond}} + (1-w) \cdot \ell^{(i)}_{\text{uncond}}\)，绕过跨模态切换令牌避免干扰相位管理。
  - 采样后，采样令牌同步追加到伴随请求，保持完全相同的自回归历史；伴随请求对外部客户端不可见。
  - 该设计吸收双分支开销与logit同步成本，使CFG吞吐保持在无CFG的80%左右。

### 6. 论文的主要结论与发现

- 基于 vLLM 的连续批处理推断管线可将多流语音语言模型的令牌处理速率提升**约两个数量级**（最高达 108 倍），同时将 MFU 从极低水平拉升至约 10%。
